Script test.lua

test.lua - A fake Pandoc filter that runs units for for pandoc-zotxt.lua.

# SYNOPSIS

pandoc --lua-filter test.lua -o /dev/null FILE

# DESCRIPTION

A fake Pandoc filter that runs units for for pandoc-zotxt.lua. Which tests are run is goverend by the tests metadata field in FILE. This field is passed to lu.LuaUnit.run. If tests is not set, runs all tests.

split_path (path)
Splits a file's path into a directory and a filename part.

Parameters:

  • path string The path to the file.

Returns:

  1. string The file's path.
  2. string The file's name.

    This function makes an educated guess given the string it's passed. It doesn't look at the filesystem. The guess is educated enough though.

copy (orig)
Copies tables recursively.

Handles metatables, recursive structures, tables as keys, and avoids the __pairs and __newindex metamethods. Copies are deep.

Parameters:

  • orig The original.

Returns:

    A copy.

Usage:

    > x = {1, 2, 3}
> y = {x, 4}
> c = copy(y)
> table.insert(x, 4)
> table.unpack(c[1])
1       2       3
read_md_file (fname)
Reads a Markdown file.

Parameters:

  • fname string Name of the file.

Returns:

  1. The parsed data, nil if an error occurred.
  2. string An error message, if applicable.
  3. number An error number. Positive numbers are OS error numbers.

    May raise an uncatchable error.
